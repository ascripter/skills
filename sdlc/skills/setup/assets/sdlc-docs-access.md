# SDLC Document Access ‚Äî slice, don't slurp

The SDLC specs under `docs/` (notably `docs/PRD.yaml`, `docs/DATA-MODEL.yaml`
and `docs/TASKS.json`) grow large enough that loading one whole burns a big
slice of an agent's context
window. They are consumed by agents with limited context, so retrieve only the
structure plus the minimum relevant slice. `Read` with `offset`/`limit` loads
**only** the requested lines ‚Äî the file's total size is irrelevant once you know
the range.

## The entrypoint: `docs/INDEX.yaml`

A **generated** location map ‚Äî never hand-edit it. It is regenerated by the
`Write|Edit` PostToolUse hook on every canonical `docs/*.yaml` or
`docs/TASKS*.json` edit ‚Äî shard edits refresh the `shards:` inventory ‚Äî (or by
running `python .claude/sdlc/docs_index.py`). It contains four blocks:

- `generated_from` ‚Äî per-file `sha256` prefix + line count (staleness guard).
- `sections` ‚Äî for each canonical doc (YAML **and** the JSON canonicals
  `TASKS.json` / `CODE-MANIFEST.json`), every top-level key ‚Üí `[start, end]`
  line range.
- `shards` ‚Äî every `docs/<PARENT>__<slug>.yaml|json` sub-artifact present,
  keyed by its parent canonical (`UX.yaml: [UX__login.yaml, ‚Ä¶]`). Use it to
  *discover* which per-surface/container/resource files exist ‚Äî then load the
  shard whole (they are small by design).
- `symbols` ‚Äî **grouped by file**; each addressable symbol is one positional row:
  `name: [start, end, kind, context, summary]` (`context` is `~` when none). The
  owning file is the group header the row sits under ‚Äî read the first two numbers
  as the `[start, end]` line range. Symbols cover DATA-MODEL `entities` + `enums`,
  PRD `FR-###` items + any `conventions.stage_dossiers` entries, and
  TASKS.json `TSK-###` tasks.

`INDEX.yaml` duplicates no field bodies; `summary` is a ‚â§120-char hint only. The
canonical truth always lives in the source file ‚Äî resolve there before relying
on it.

## Retrieval protocol

1. **Never read `PRD.yaml`, `DATA-MODEL.yaml`, or `TASKS.json` in full.** Load
   `docs/INDEX.yaml` first.
2. **Resolve by symbol, read by range.** Look the name up in `symbols`, then
   `Read(file, offset=start, limit=end-start+1)`. Use `summary` to pick the right
   symbol without opening several.
3. **Resolve a top-level block by its `sections` range.** To read one block of a
   doc (e.g. PRD `conventions`, DATA-MODEL `cross_references`), slice the
   `[start, end]` range listed under `sections.<file>.<key>` ‚Äî don't load the file
   whole to find it.
4. **Follow cross-references lazily.** Only resolve a linked entity when the task
   needs its fields. To traverse relationships, slice the `composition` /
   `cross_references` sections (via their `sections` ranges) ‚Äî do not load
   `entities` whole to find a link.
5. **Never inline large repeated blocks** into prompts or other artifacts; reference
   a symbol by name + range.
6. **Self-heal a stale range.** If a sliced range does not begin with the expected
   `<name>:` anchor, the index lagged an edit: relocate with `Grep`
   (`^\s*<name>:` for entities/enums/dossiers; `"<FR-id>:` for an FR), `Read`, then
   regenerate with `python .claude/sdlc/docs_index.py`.

## Power tool

`python .claude/sdlc/docs_index.py --show <symbol>` prints exactly one symbol's
slice (`<symbol>` = an entity/enum name, an `FR-###` id, a `TSK-###` id, or a
`conventions.stage_dossiers` map key). Equivalent to looking the symbol up in
`INDEX.yaml` and reading its range.

## Maintaining the index

- `INDEX.yaml` is a derived artifact ‚Äî edit the canonical source; the hook refreshes it.
- The generator (`.claude/sdlc/docs_index.py`) content-indexes the canonical
  docs only. Per-surface / per-container sub-artifacts (`UX__*.yaml`,
  `ARCH__*.yaml`, `API__*.yaml`, `TASKS__*.json`), versioned snapshots, and
  `*_draft` files are not sectioned ‚Äî they are independent documents small
  enough to load whole ‚Äî but present shards ARE enumerated in the `shards:`
  block for discovery, and writing one refreshes the index.
- Do **not** split `PRD.yaml` / `DATA-MODEL.yaml` / `TASKS.json`: range-based
  slicing already gives
  free partial reads, and the `entities` graph (`composition`, `cross_references`)
  is a whole-file relation that splitting would fragment. (UX/ARCH/API/TASKS are
  sharded because their surfaces/containers/resources are independent documents.)
